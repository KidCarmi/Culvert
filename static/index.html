Automatically import blocked domains from a remote text feed (one domain per line, # comments allowed). New entries are merged — manually-added entries are never removed. This is separate from URL Categories: domains imported here are fully blocked regardless of policy rules.

Free public feeds: https://urlhaus.abuse.ch/downloads/text/ (malware) · https://raw.githubusercontent.com/StevenBlack/hosts/master/hosts (ads/tracking)

How requests are evaluated: Blocklist → Threat Feeds → Policy Rules (priority order) → Default Action
Blocklist entries are checked first and hard-block regardless of policy. Threat feeds (URLhaus, OpenPhish) are checked during scanning. Policy rules are evaluated in priority order — first match wins.

🔑 Install Root CA — Instructions
Windows:
  1. Double-click culvert-ca.pem → Install Certificate
  2. Select "Local Machine" → Place in "Trusted Root CAs"

macOS:
  1. Double-click culvert-ca.pem → Keychain Access
  2. Find "Culvert Root CA" → Get Info → Trust → Always Trust

Linux (Ubuntu/Debian):
  sudo cp culvert-ca.pem /usr/local/share/ca-certificates/culvert.crt
  sudo update-ca-certificates

Firefox (all platforms):
  Settings → Privacy → View Certificates → Authorities → Import

🔑 Proxy Authentication Mode
🔒
Credentials Required
Proxy requires Proxy-Authorization. Clients must authenticate before any traffic is forwarded.

Credentials Required — Proxy port demands authentication (407). Use when all clients authenticate via local/LDAP/OIDC credentials.
Open / Policy-Only — Proxy accepts all clients without credentials. Identity is set to unauth for unauthenticated traffic. Policy rules control access — create a rule with Auth Source = unauth to allow or block unauthenticated users. Recommended when using a PAC file with PBAC.
